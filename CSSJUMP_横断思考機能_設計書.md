# CSSJUMP 横断思考機能 設計書

**作成日**: 2026-04-16
**ステータス**: アイデア段階（ブレスト用メモ）
**目的**: Copilot/Claude Code が使えない環境でも、CSSJUMP 単体で「横断的に考える」機能を持たせる

---

## 🎯 ゴール

CSSJUMP に以下の2つの要素を追加する：

1. **横断性**：複数ファイル（HTML / CSS / JS / PHP / メモ）を**同時に読んで**答えを出す
2. **思考力**：段階的に考えて、根拠付きの回答を返す

**要するに Claude Code がやっている仕事を、CSSJUMP の中に閉じ込める**。

---

## 💡 なぜ必要か

| 現状の CSSJUMP | 足りないもの |
|----------------|--------------|
| Ctrl+I の AI 質問（単一ファイル or 選択範囲のみ） | 🚫 他ファイルの文脈を見ない |
| Alt+クリックでジャンプ（単発の移動） | 🚫 関連ファイル群を一望できない |
| メモ検索（Ctrl+Enter） | 🚫 コードとメモを横断しない |

→ **「このボタンが押された時、何が連動して動いてる？」に答えられない**
→ これこそ横断思考が欲しい場面

---

## 🗺 機能設計（3段階）

### Lv1：関連ファイル自動収集 🟢（小）

**トリガー**：`Ctrl+Shift+I`（新ショートカット案）

**動作**：
1. 開いているファイルと「同じフォルダ / 同じセレクタを持つ / import されている」ファイルを収集
2. プロジェクトから **関連度の高い上位5〜10ファイル** を選ぶ
3. 全部まとめて AI に投げる
4. 回答を右側パネルに表示

**関連度スコアの判定例**：
- 同一ディレクトリ: +3
- `<link>` / `import` で参照されている: +5
- 同じクラス名が使われている: +4
- 同じ関数名が呼ばれている: +4

---

### Lv2：セレクタ・関数の横断追跡 🟡（中）

**トリガー**：要素を選択 → 右クリック「横断的に解析」

**動作**：
1. 選択されたセレクタ（例: `.hero`）や関数名（例: `get_hero_data`）を**全ファイルから grep**
2. 使用箇所を**すべて抽出**（HTML / CSS / JS / PHP / メモ 全部）
3. それらの**前後10行を文脈として**AIに送る
4. 「この要素に関係する全体像」を AI に整理させる

**出力イメージ**：
```
🔍 .hero の横断解析結果

📄 定義元：
  - style.css:42 （PC用スタイル）
  - style.css:312 （@media モバイル用）

📄 使用箇所（HTML/PHP）：
  - index.html:15
  - header.php:8
  - front-page.php:22

📄 関連JS：
  - hero.js:12（クリックイベント）

📖 関連メモ：
  - 01_memo.md:845（.hero の設計理由）

💡 AI総評：
  この .hero は3つのページで使われていて、
  モバイル時はpadding が半分になる設計。
  hero.js で onClick ハンドラが付いている。
  改善候補：@media の重複定義が1箇所あり（line 328）。
```

---

### Lv3：プロジェクト全体サマリ＋質問 🔴（大）

**トリガー**：コマンドパレット「プロジェクト全体に質問」

**動作**：
1. 初回：全ファイルをスキャン → **各ファイルの要約（100字程度）をキャッシュ**（`.cssjumper_cache/`）
2. 質問時：全要約をまず AI に渡す → **AI が「どのファイルを深く読むべきか」を判断**
3. 選ばれたファイル群だけを**本文ごと**AI に送って最終回答
4. ファイル変更時は差分だけ再要約（高速）

**これが事実上の「社内版 Claude Code」になる**。

## 🧠 「よく考える」の実装

### 裏プロンプト戦略（必須）

```
あなたはシニアWeb開発者です。以下の手順で回答してください：

【Step 1】与えられたファイル群を読み、関係性をマップ化する
【Step 2】ユーザーの質問に関係する要素を特定する
【Step 3】複数ファイルの矛盾・重複・設計上の問題を探す
【Step 4】根拠をコード引用付きで示しながら回答する

回答フォーマット：
① 現状分析（3行以内）
② 問題点（あれば箇条書き、根拠コード引用）
③ 改善案（優先度付き）
④ 影響範囲（どのファイルを触るべきか）
```

### モデル切り替え機能（重要）🔥

**アイデア**：用途によって AI モデルを切り替える UI を設定に追加

| 用途 | 推奨モデル | 理由 |
|------|-----------|------|
| 通常の質問（Ctrl+I） | Claude Haiku 4.5 or Gemini Flash | 速い・安い |
| Lv1 横断（5ファイル程度） | Claude Sonnet 4.6 | バランス型 |
| Lv2 横断（詳細追跡） | Claude Sonnet 4.6 | 精度重視 |
| Lv3 プロジェクト全体解析 | **Claude Opus 4.6** | 最高の思考力・長文処理 |
| コード自動要約（キャッシュ用） | Gemini Flash Lite | 大量処理に対応 |

**実装**：
- `settings.json` に `cssToHtmlJumper.modelByTask` を追加
  ```json
  {
    "cssToHtmlJumper.modelByTask": {
      "quick": "claude-haiku-4-5-20251001",
      "crossFile": "claude-sonnet-4-6",
      "deepAnalysis": "claude-opus-4-6",
      "summarize": "gemini-2.5-flash-lite"
    }
  }
  ```
- 実行時に UI で切り替えボタン（「🐇 速 / 🦅 標準 / 🧠 深く考える」）
- **「🧠 深く考える」モードは Opus 4.6 に自動切替**

## 📦 技術要素

| 要素 | 実装方針 |
|------|----------|
| ファイル収集 | VS Code API `workspace.findFiles` |
| grep | `ripgrep`（VS Code 内蔵）を spawn |
| 要約キャッシュ | `.cssjumper_cache/summaries.json` |
| トークン管理 | 送信前にファイルサイズチェック、超えたら要約版を送る |
| API切替 | `claudeApiKey` / `geminiApiKey` を用途別に使い分け |
| 結果表示 | 既存の右側分割パネル流用（Ctrl+I と同じUI） |

---

## 🛡 会社利用時の安全設計

1. **送信前プレビュー**：「このファイルがAIに送られます」と確認ダイアログ
2. **ブラックリスト**：`.env` / `secrets.*` / `node_modules/` は絶対送らない
3. **ローカルLLM 対応**：Ollama エンドポイントに切り替えられる設定を用意
4. **ログ機能**：送信履歴を `.cssjumper_cache/sent_log.md` に記録（監査用）
5. **機能OFFスイッチ**：settings.json 1行で横断機能を全部停止できる

## 📖 メモ（ブレストで出た論点）

- Copilot禁止の会社でも、**個人のAPIキーで直接叩く**なら許される場合あり
- 「Claude Code と何が違うの？」→ **プロジェクト特化型、CSS/HTML/PHPに最適化、メモ連携あり**
- **深く考えたい時は Opus 4.6**、普段は Haiku/Flash でコスト抑制 ← これが肝
- 研修先の会社規模（中小・大手案件なし）なら、この機能も導入できる可能性高い
